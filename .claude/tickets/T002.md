# T002 — Kafka Topic + Contract Definitions

## Why this ticket exists
Kafka is the nervous system of this platform — seven different services talk to each other exclusively through events. Without a written contract for each event's shape, a .NET service and a Java service that share the same topic will silently diverge: one adds a field, the other doesn't know it exists, and the bug only surfaces at runtime under load. This ticket pins the shape of every Kafka message as a JSON Schema document so any service — regardless of language — can validate what it sends and receives before a single line of handler code is written.

## What to build
- `/contracts/kafka/user.verified.schema.json` — fields: `event_id`, `user_id`, `gender`, `verified_at`
- `/contracts/kafka/connection.accepted.schema.json` — fields: `event_id`, `connection_id`, `requester_id`, `recipient_id`
- `/contracts/kafka/connection.expired.schema.json` — fields: `event_id`, `connection_id`, `requester_id`
- `/contracts/kafka/chat.closed.schema.json` — fields: `event_id`, `chat_id`, `connection_id`, `user_a_id`, `user_b_id`, `closed_reason`
- `/contracts/kafka/rating.submitted.schema.json` — fields: `event_id`, `rating_id`, `rater_id`, `rated_id`, `tags` (array)
- `/contracts/kafka/trust.score.updated.schema.json` — fields: `event_id`, `user_id`, `new_score`, `badge_level`, `request_limit`
- `/contracts/kafka/report.filed.schema.json` — fields: `event_id`, `report_id`, `reporter_id`, `reported_id`, `reason`, `report_count_for_target`
- `/contracts/kafka/README.md` — topic name, partitions (3), retention (7 days), producer, consumers per topic

## Technical specification

Create one JSON schema file per Kafka topic in `/contracts/kafka/`. Each file defines the message shape. Use JSON Schema draft-07.

Topics to define (see REQUIREMENTS.md for fields):
- `user.verified.schema.json`
- `connection.accepted.schema.json`
- `connection.expired.schema.json`
- `chat.closed.schema.json`
- `rating.submitted.schema.json`

- `report.filed.schema.json`

Also create `/contracts/kafka/README.md` documenting topic names, partitions (default 3), retention (7 days), and which services produce/consume each.

## Acceptance criteria

## Patterns demonstrated
**Event-Driven Architecture** — topics carry past-tense facts (`connection.accepted`, not `open-chat-please`), decoupling the sender from the receiver entirely.
**Schema-first design** — writing the contract before writing the code forces agreement between producer and consumer teams before either line is committed.
**Outbox Pattern (prerequisite)** — `event_id` (UUID) is a required field on every schema; this is what makes consumer-side idempotency possible — without it, at-least-once delivery has no safe deduplication key.
**Idempotency** — the `event_id` field enables every Kafka consumer to maintain a `processed_events` table and skip duplicate deliveries safely.

## Agent implementation notes
- Key files to create: 7 JSON schema files + 1 README in `/contracts/kafka/`
- All schemas use `"$schema": "http://json-schema.org/draft-07/schema#"`
- All field names are `snake_case` — no camelCase anywhere in Kafka payloads
- Every schema MUST include `event_id` as a required string (UUID format) — this is the deduplication key consumed by idempotency logic in every subscriber
- `closed_reason` in `chat.closed` should be an enum: `"met_successfully"` | `"timeout"` | `"admin_closed"`
- `badge_level` in `trust.score.updated` should be an enum: `"trusted"` | `"standard"` | `"restricted"`
- `tags` in `rating.submitted` is an array of strings — use an enum constraint for valid tag values (positive: `friendly`, `respectful`, `on_time`, `good_communication`, `would_travel_again`; negative: `didnt_show_up`, `fake_destination`, `abusive`, `spam`, `asked_for_money`)
- README table columns: Topic | Producer | Consumers | Partitions | Retention
- Do NOT add Kafka broker config here — that is T004. This ticket is schema definitions only.
- Do NOT embed these schemas inside service code — they live in `/contracts/kafka/` as the single source of truth.
